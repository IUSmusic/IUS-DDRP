# IUS DRRP Desktop characteristics

## Product identity

**Official name:** IUS DRRP Desktop 
**Category:** desktop hi-fi recorder / playback / offline radio workstation 
**Primary use case:** studio and home playback, monitoring, recording, and offline radio listening

## Core characteristics

- Mains-powered desktop unit with internal backup battery continuity target
- Integrated playback, radio, and record modes
- 15-band manual EQ with direct physical-control intent
- EQ graph as the default visual state
- 5-track DAW-style monitor overlay triggered by interaction
- Local-first media / project workflow
- Tactile control-first UX rather than touchscreen-first UX
- Built-in speaker architecture with side + top spatial driver direction
- Dedicated headphone / monitor output requirement
- Stereo line input and output requirement
- Stereo digital sync target (S/PDIF in / out recommended in builder docs)

## UX requirements

- Immediate mode switching between Player / Radio / Record
- No confusing hidden states between playback and monitoring
- Minimal but technically credible graph design
- Clear source / quality / record status visibility
- Track overlay must be useful, not decorative
- Presets must remain modest and believable
- Browser demo should feel responsive even before embedded-audio integration

## Product-level engineering intent

- Offline operation for playback, projects, and radio
- A dedicated audio path better than generic onboard audio
- A path toward low-latency embedded audio rather than a general-purpose desktop app
- A path toward a productisable carrier board / power system / amplifier design
